# 司法拍卖信息定期抓取脚本方案

## 目标

建立一个可以长期运行的自动化脚本，用于定期抓取淘宝司法拍卖上的房产拍卖信息，并筛选出符合条件的标的。

当前讨论的核心筛选条件：

- 来源网站：淘宝司法拍卖 / 阿里资产
- 地区：上海市浦东新区
- 标的类型：房产，优先住宅用房，也可扩展到商业用房、工业用房、其他用房
- 时间范围：未来 5 天内即将开拍或正在进行的拍卖
- 金额门槛：起拍价、当前价或变卖价在 1000 万元以上
- 输出内容：地址、价格、评估价/市场价、开拍时间、报名人数、围观人数、状态、详情链接

## 推荐运行环境

建议把脚本放在 Linux 电脑上长期运行：

```text
Linux 电脑：ecko@192.168.10.45
```

原因：

- Linux 可以后台运行，不影响 Mac 日常使用
- 已经安装并验证了 Node.js 和 Playwright
- 可以用定时任务自动执行
- 适合保存历史数据、日志和每日报告

当前已经验证过的能力：

```text
Node.js：已安装到用户目录
Playwright：已安装
Google Chrome：已存在，并可被 Playwright 调用
网页读取：已验证可以打开网页并读取页面标题和正文
```

## 脚本执行流程

每天固定时间运行一次，流程如下：

1. 打开淘宝司法拍卖页面
2. 进入房产/住宅用房列表
3. 筛选上海市、浦东新区
4. 读取未来 5 天内的拍卖标的
5. 筛选价格超过 1000 万元的标的
6. 提取结构化字段
7. 和上一次抓取结果对比
8. 生成 Markdown/CSV/JSON 文件
9. 记录日志

## 建议提取字段

每个标的建议保存这些字段：

```text
title           标的名称/地址
district        所在区域
price_type      起拍价/当前价/变卖价
price_wan       价格，单位万元
valuation_wan   评估价或市场价，单位万元
start_time      开拍时间
end_time        预计结束时间，如有
status          即将开始/正在进行/变卖等
watch_count     围观人数
signup_count    报名人数
loan_supported  是否支持贷款
url             详情页链接
captured_at     抓取时间
```

## 定时运行方式

### 方案一：cron

适合简单定时，例如每天早上 9 点执行：

```cron
0 9 * * * cd /path/to/sifapaimai && /home/ecko/.local/node/bin/node scripts/fetch-auctions.js >> logs/cron.log 2>&1
```

### 方案二：systemd timer

适合更稳定的长期任务，可以记录状态、失败重试和日志。后续如果要正式长期运行，建议使用 systemd timer。

## Token 消耗说明

脚本做成独立定时任务以后，每次自动执行时不需要消耗 Codex token。

原因是：

```text
Linux 定时器触发脚本
脚本自动打开网页/读取数据
脚本自动筛选和对比
脚本自动保存结果
```

这些动作都在 Linux 本机完成，不需要 Codex 实时参与。

会消耗 token 的情况主要是：

- 让 Codex 修改脚本
- 让 Codex 排查错误
- 让 Codex 分析抓取结果
- 让 Codex 增加新的筛选规则
- 让 Codex 生成更复杂的报告

## 注意事项

- 当前抓取的是公开展示信息，不需要登录淘宝账号
- 报名、缴纳保证金、出价等动作需要登录账号，脚本不应自动执行这些操作
- 页面结构可能变化，所以脚本需要保留日志和错误截图
- 定时任务建议低频运行，例如每天 1-2 次，不做高频请求
- 价格字段需要统一单位，建议全部转换成“万元”
- 时间字段需要补全年份，避免跨年时判断错误
